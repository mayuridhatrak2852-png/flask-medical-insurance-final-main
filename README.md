# Medical Insurance Charges Prediction

A Flask web application that predicts medical insurance charges using a Linear Regression model.

## Project Structure

```
├── app.py                  # Flask app (routes + prediction)
├── train_model.py          # Train model & save pickle + JSON
├── medical_insurance.csv   # Training dataset
├── Linear_Model.pkl        # Pre-trained model (generated by train_model.py)
├── project_data.json       # Model metadata (generated by train_model.py)
├── requirements.txt        # Dependencies
├── templates/index.html    # Form + result page
├── static/css/style.css    # Styling
├── .gitignore
└── README.md
```

## Technologies Used

- Python
- Flask
- Scikit-learn
- Pandas
- NumPy
- Gunicorn

## Local Setup

```bash
# Clone the repository
git clone https://github.com/ajeetkumarAI/flask-medical-insurance-final.git

cd flask-medical-insurance

# Create virtual environment
python -m venv venv

# Activate virtual environment
venv\Scripts\activate             # Windows
# source venv/bin/activate        # Linux / macOS


# Install dependencies
pip install -r requirements.txt

# Train the model (generates Linear_Model.pkl and project_data.json)
python train_model.py

# Run the app
python app.py
```

Open http://localhost:8000 in your browser.


### sklearn warning (X does not have valid feature names) is harmless — it just means we pass a numpy array instead of a DataFrame to predict(). It doesn't affect the results at all.

### 200 = file sent, 304 = file hasn't changed, use @ cache. Both are perfectly normal.

## Azure Deployment

1. Create a GitHub repo and push this code to `main`
2. Create Azure Web App (Python 3.12, Linux)
3. In Azure Portal → Deployment Center → Connect your GitHub repo
4. Set Startup Command in Configuration → General Settings:
   ```
   gunicorn --bind=0.0.0.0:8000 --timeout 600 app:app
   ```
